# Stellar Burgers — обзор автоматизации тестирования

Этот документ описывает подход к автоматизации тестирования
в проекте Stellar Burgers и связывает unit-, API- и UI-уровни
в единую пирамиду тестирования.

Цель автоматизации — проверить функциональность продукта
на разных уровнях с минимальной избыточностью и высокой устойчивостью тестов.

В рамках проекта реализовано:
- ~30–40 unit-тестов,
- ~20–30 API-тестов,
- ~15–20 e2e UI-сценариев.

---

## Подход и мотивация

Автоматизация построена по принципу пирамиды тестирования:

- unit-тесты — проверка бизнес-логики;
- API-тесты — проверка контрактов и поведения сервисов;
- UI-тесты — проверка ключевых пользовательских сценариев.

Каждый уровень решает свою задачу и не дублирует проверки других уровней.

---

## Unit-тестирование

Unit-тесты используются для:
- проверки бизнес-логики;
- валидации граничных условий;
- проверки обработки исключений.

Тесты изолированы от UI и внешних сервисов,
что позволяет быстро получать обратную связь при изменениях кода.

Подробности: [stellar-burgers-unit.md](./stellar-burgers-unit)

---

## API-автоматизация

API-тесты используются для:
- проверки REST API без участия UI;
- валидации структуры и данных ответов;
- проверки позитивных и негативных сценариев.

API-уровень позволяет выявлять ошибки
раньше и дешевле, чем UI-автотесты.

[Подробности: stellar-burgers-api.md](./stellar-burgers-api.md)

---

## UI-автоматизация

UI-автотесты покрывают:
- ключевые пользовательские сценарии;
- критические бизнес-флоу.

UI-тесты намеренно ограничены по количеству
и не используются для проверки логики,
уже покрытой unit- и API-тестами.

Подробности: [stellar-burgers-ui.md](./stellar-burgers-ui.md)

---

## Итог

Автоматизация Stellar Burgers построена как система,
а не набор разрозненных тестов.

Подход позволяет:
- сократить количество хрупких UI-тестов;
- быстрее выявлять дефекты;
- поддерживать тесты при изменениях продукта.
